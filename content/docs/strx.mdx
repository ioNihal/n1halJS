## Quick Start

**@n1hal/strx** provides simple, dependency free utilities for common *string* manipulation tasks.
It includes full **TypeScript declarations**, so your editor automatically infers *types*, no manual typing needed.
Each function is **documented with JSDoc**, giving you instant hover time *descriptions* and *examples* in you code editor itself.

The library is **intentionally minimal**, you can even open the source and *copy-paste* any function without setup or dependencies.
Every helper is written in **clean**, readable **JavaScript** for effortless *understanding* and use.

## Usage Example

You can import individual functions for tree-shaking and minimal bundle size, or use the **default export** to access all functions at once. All functions include **TypeScript type declarations** for autocompletion and type safety.

### Named Imports

<Tabs items={['Javascript', 'Typescript']}>
  <Tab value="Javascript">
    ```js 
    import { capitalize, camelCase, slugify } from "@n1hal/strx";
    
    const capitalizedText = capitalize("hello world"); // "Hello world"
    const camelCasedText = camelCase("hello world example"); // "helloWorldExample"
    const sluggedText = slugify("Hello World! Café"); // "hello-world-cafe"
    ```
  </Tab>
  <Tab value="Typescript">
    ```ts
    import { capitalize, camelCase, slugify } from "@n1hal/strx";
    
    const capitalizedText: string = capitalize("hello world"); // "Hello world"
    const camelCasedText: string = camelCase("hello world example"); // "helloWorldExample"
    const sluggedText: string = slugify("Hello World! Café"); // "hello-world-cafe"
    ```
  </Tab>
</Tabs>

<Tabs items={['Javascript', 'Typescript']}>

#### sentenceCase()
Converts a string to **sentence case** by capitalizing the **first letter**.  
If the string does **not already end** with a period (`.`), exclamation mark (`!`), or question mark (`?`), it will **automatically add a period** (`.`) at the end.

| Parameter | Type | Default | Description |
|------------|------|----------|-------------|
| `str` | `string` | `""` | The input string to convert to sentence case |

**Returns:** `string`

<Tabs items={["Javascript", "TypeScript"]}>
  <Tab value="JavaScript">
    ```js
    sentenceCase("hello world"); // "Hello world."
    sentenceCase("what is this?"); // "What is this?"
    sentenceCase("this is fine!"); // "This is fine!"
    ```
  </Tab> 
  <Tab value="TypeScript">
    ```ts
    const result: string = sentenceCase("hello world"); // "Hello world."
    const resultTwo: string = sentenceCase("what is this?"); // "What is this?"
    const resultThree: string = sentenceCase("this is fine!"); // "This is fine!"
    ```
  </Tab>
</Tabs>

### 4. Special Helpers

#### slugify()
Creates a **URL-friendly slug** from a string by replacing spaces and special characters with a separator (default `-`).  
It can optionally preserve Unicode letters and convert the result to lowercase.

| Parameter | Type | Default | Description |
|------------|------|----------|-------------|
| `input` | `string` | `""` | The input string to convert into a slug |
| `options.separator` | `string` | `"-"` | Character to use as the separator in the slug |
| `options.lowercase` | `boolean` | `true` | Convert the slug to lowercase if true |
| `options.preserveUnicode` | `boolean` | `false` | Keep Unicode characters if true |

**Returns:** `string`

<Tabs items={["Javascript", "TypeScript"]}>
  <Tab value="JavaScript">
    ```js
    slugify("Hello World! Café"); // "hello-world-cafe"
    slugify("Custom Separator Example", { separator: "_", lowercase: false }); // "Custom_Separator_Example"
    ```
  </Tab> 
  <Tab value="TypeScript">
    ```ts
    const result: string = slugify("Hello World! Café"); // "hello-world-cafe"
    const resultTwo: string = slugify("Custom Separator Example", { separator: "_", lowercase: false }); // "Custom_Separator_Example"
    ```
  </Tab>
</Tabs>

#### pluralize()
Pluralizes a simple English noun based on basic rules.  
If the count is 1, the word is returned as-is. Otherwise, it applies common pluralization rules like adding `s`, `es`, or replacing `y` with `ies`.

| Parameter | Type | Default | Description |
|------------|------|----------|-------------|
| `word` | `string` | `""` | The singular noun to pluralize |
| `count` | `number` | `2` | The count of items; if 1, the singular form is returned |

**Returns:** `string`

<Tabs items={["Javascript", "TypeScript"]}>
  <Tab value="JavaScript">
    ```js
    pluralize("cat", 1); // "cat"
    pluralize("cat", 3); // "cats"
    pluralize("box", 2); // "boxes"
    pluralize("baby", 5); // "babies"
    ```
  </Tab> 
  <Tab value="TypeScript">
    ```ts
    const result: string = pluralize("cat", 1); // "cat"
    const resultTwo: string = pluralize("cat", 3); // "cats"
    const resultThree: string = pluralize("box", 2); // "boxes"
    const resultFour: string = pluralize("baby", 5); // "babies"
    ```
  </Tab>
</Tabs>
